# 仓库物料管理系统

一个基于 C/C++ 实现的控制台仓库管理系统，完成于数据结构课程设计。系统面向万级物料数据管理场景，支持货物入库、出库、库存查询、分类浏览、供应商管理、报表生成和文件持久化等功能。

## 项目简介

本项目围绕仓库物料的日常管理流程展开，设计了货物信息、出库记录、供应商信息和管理员信息等数据结构，并通过文件读写实现数据持久化。系统使用链表组织动态数据，适合处理频繁新增、删除和修改的仓库记录；在查询阶段将链表转换为数组，结合快速排序与二分查找提高万级数据下的检索效率。

## 主要功能

- 用户与权限管理：支持经理登录、管理员登录与注册、管理员信息浏览、添加和删除。
- 货物入库：录入货物名称、体积、重量、价格、保质期、采购人、验收人、入库日期和生产厂家等信息。
- 自动编码：根据入库日期、存放区域、货物名称首字母和重复序号生成唯一货物编码。
- 货物出库：支持按货物编号出库，以及按货物名称和数量批量出库。
- 出库记录维护：记录出库货物编号、名称、出库日期、去向和联系方式。
- 货物查询与浏览：支持按编号、名称、区域、厂家等字段查询，并支持按名称和入库时间排序浏览。
- 库存盘点：支持删除已出库货物信息，避免已出库物料继续占用库存空间。
- 过期检查：根据当前日期和保质期筛选报废/过期货物。
- 供应商管理：支持供应商信息查询、浏览和添加。
- 报表生成：可按年份导出入库报表和出库报表。
- 文件持久化：货物、出库记录、供应商、管理员等信息均通过文本文件保存。

## 技术实现

### 数据结构设计

系统主要使用以下链表结构组织数据：

- `goods`：货物信息节点，包含编码、名称、体积、重量、价格、保质期、采购人、验收人、存放地点、入库日期、出库日期和生产厂家等字段。
- `Outgood`：出库货物节点，记录出库货物编号、名称、出库时间、去向和联系方式。
- `Pro`：供应商节点，记录供应商名称与联系方式。
- `User`：管理员节点，记录管理员账号、密码、联系方式和地址。

### 查询优化

货物数据以链表形式存储，便于动态插入和删除。但单链表不适合高频查询，因此系统在查询阶段将链表转换为数组：

1. 遍历链表，将货物节点指针写入数组。
2. 使用 `qsort` 对数组按照编号、名称、区域或厂家排序。
3. 使用二分查找定位目标字段的匹配范围。
4. 查询完成后释放临时数组。

这种方式兼顾了链表的动态管理能力和数组在排序、查找上的效率优势。

### 自动编码策略

货物编码由以下部分组成：

```text
入库日期 + 区域码 + 名称首字母 + 重复序号
```

其中，区域码根据货物体积和重量划分为 A/B/C 三类；名称首字母取英文货物名称中各单词首字母，不足三位用 `_` 补齐；重复序号采用最小空缺编号分配策略，避免删除旧记录后编号空间浪费。

### 文件读写与数据清洗

系统运行时从文本文件中读取货物、出库记录和供应商信息，并在入库、出库、删除、添加供应商等操作后写回文件。针对真实数据中可能出现的字段缺失、空格、制表符等问题，代码中加入了字符串清洗和格式兼容处理，提升了系统对真实数据的鲁棒性。
