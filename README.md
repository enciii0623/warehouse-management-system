# C/C++ 库存管理系统

一个控制台库存管理系统，支持用户登录与权限管理、货物入库、货物出库、库存查询、分类浏览、供应商管理、库存盘点、过期检查、报表生成和文件持久化等功能。项目的核心数据结构设计、功能逻辑实现和系统联调由小组独立完成。

## 主要功能

### 用户登录与权限管理

系统设置经理和管理员两类角色。经理账号用于管理员信息维护，支持浏览管理员信息、添加管理员和删除管理员；管理员账号用于进入库存管理主功能，包括入库、出库、货物管理、供应商管理、库存盘点和报表生成等操作。

### 货物入库

入库模块用于录入新货物信息，包括货物名称、体积、重量、价格、保质期、采购人、验收人、入库日期和生产厂家等字段。录入完成后，系统会根据货物信息自动生成唯一编码和存放位置，并将新货物追加到货物链表和货物信息文件中。

### 自动编码

系统根据入库日期、货物类别、名称首字母和重复序号生成货物编码。类别码由货物体积和重量划分为 A/B/C 三类；名称首字母取英文货物名称中各单词首字母，不足三位用 `_` 补齐；重复序号用于区分同名货物。

### 货物出库

出库模块支持两种方式：按货物编号出库，以及按货物名称和数量批量出库。出库时系统会检查货物是否已经出库，避免重复操作；出库成功后会更新货物的出库日期，并将货物编号、名称、出库时间、去向和联系方式写入出库记录文件。

### 库存查询与分类浏览

货物管理模块支持按编号、名称、区域和厂家等字段查询货物信息，也支持按名称和入库时间进行分类浏览。为提高查询效率，系统在查询阶段将链表转换为数组，并结合快速排序和二分查找定位匹配结果。

### 供应商管理

供应商管理模块支持供应商信息查询、浏览和添加。系统从供应商文件中读取供应商名称与联系方式，查询时根据名称匹配供应商信息，添加供应商后会同步写回文件。

### 库存盘点

库存盘点模块用于维护库存状态，支持删除已出库货物信息，避免已出库货物继续占用库存空间。系统会遍历货物链表，根据出库日期判断货物状态，并在删除后同步更新货物信息文件。

### 过期检查

系统根据货物入库日期、保质期和用户输入的当前日期判断货物是否过期，用于辅助筛选报废或需处理的货物。

### 报表生成

报表模块支持按年份生成入库报表和出库报表。用户输入年份后，系统遍历对应链表，筛选该年份的入库或出库记录，并导出为文本报表文件，便于后续查看和统计。

### 文件持久化

系统通过文本文件保存货物、出库记录、供应商和管理员信息。程序启动时从文件读取数据并构建链表，执行入库、出库、删除、添加供应商等操作后再写回文件，实现数据在多次运行之间的保留。

## 数据结构与实现

系统使用链表组织货物、出库记录、供应商和管理员等动态数据，便于进行新增、删除和修改操作。为避免尾部插入时反复遍历链表，程序维护尾指针以提高插入效率。

查询时，系统将链表数据转换为数组，利用数组在排序和查找上的优势，结合快速排序与二分查找提升检索效率。文件读取部分采用逐行读取和动态内存分配方式，能够处理较大规模数据，并对名称中包含空格、制表符等情况进行兼容处理。

## 自动编码规则

货物编码由以下部分组成：

```text
货物编码 = 入库日期 + 类别码 + 名称首字母 + 重复序号
```

其中：

- 类别码根据货物体积和重量划分为 A/B/C。
- 名称首字母取英文货物名称中各单词首字母，不足三位用 `_` 补齐。
- 重复序号用于区分同名货物，并采用最小空缺编号分配策略。

示例：

```text
20160101ACD_000
```

表示该货物入库日期为 2016-01-01，类别码为 A，货物名称首字母为 `CD_`，重复序号为 `000`。

## 运行方式

1. 将四个 txt 数据文件与程序运行目录放在同一级。
2. 在 Windows 环境下使用 Dev-C++、Visual Studio 或 MinGW 编译运行 `src/warehouse_management.cpp`。
3. 按控制台菜单提示进行登录、入库、出库、查询等操作。

## 数据说明

课程设计过程中使用万级真实仓库数据进行预处理和系统测试。由于原始数据涉及人员、联系方式和供应商等信息，不便在公开仓库中直接提供，因此本仓库仅保留脱敏后的示例数据，用于演示程序的文件读写、查询和出入库流程。
